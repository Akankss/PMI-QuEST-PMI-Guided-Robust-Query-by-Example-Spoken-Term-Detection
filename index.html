step 03 of 07
Select PMI bigrams
PMI measures how much more adjacent token pairs co-occur than expected under independence. Only pairs exceeding threshold τ = 0.5 enter the index.
PMI formula
PMI(a,b) = log   p(a,b) / (p(a)·p(b))

// p(a,b) = adjacent-pair corpus frequency
// PMI > 0 : appear together more than chance
// PMI ≤ 0 : accidental — discarded

𝒷_τ = { (a,b) ∈ 𝒱² : PMI(a,b) > τ }

step 04 of 07
Build the PMI-TF-IDF vector
Concatenate unigram TF-IDF weights with α-scaled bigram weights, then L2-normalise. The resulting vector encodes both what tokens appear (unigrams) and in what order (bigrams).
Unigram part u_q — TF-IDF weights
w_uni(t, s) = (n_t(x) / |s(x)|) · log(N / (1+d_t))

token 271 (/w/) : 1/5 × 2.3 = 0.46
token 089 (/ʌ/) : 1/5 × 3.4 = 0.68 ← high IDF, rare
token 314 (/n/) : 1/5 × 2.8 = 0.56
token 452 (/d/) : 1/5 × 2.1 = 0.42
token 177 (/ɚ/) : 1/5 × 3.1 = 0.62
Bigram part b_q — α = 1.0
w_bi((a,b), s) = (n_ab(x)/(|s|-1)) · log(N/(1+d_ab))

(089→314) : 1/4 × 3.4 = 0.85 × α = 0.85
(314→452) : 1/4 × 2.8 = 0.70 × α = 0.70
(452→177) : 1/4 × 3.1 = 0.78 × α = 0.78
Combined vector (unigrams ‖ bigrams → L2-normalise)
raw = [0.46, 0.68, 0.56, 0.42, 0.62 | 0.85, 0.70, 0.78]
‖raw‖ = √(0.46²+0.68²+0.56²+0.42²+0.62²+0.85²+0.70²+0.78²)
       = √(0.21+0.46+0.31+0.18+0.38+0.72+0.49+0.61) = √3.36 ≈ 1.83

v_q ≈ [0.25, 0.37, 0.31, 0.23, 0.34, 0.46, 0.38, 0.43]
The bigram dimensions (089→314), (314→452), (452→177) carry the highest weights in v_q. Documents that share the /ʌ/→/n/→/d/→/ɚ/ transition will be strongly pulled toward the query in HNSW space.

step 06 of 07
Smith-Waterman reranking
SW local alignment enforces full token-sequence order. Applied to the top-8 HNSW candidates. Score = max H(i,j) / m where m = max(|s_q|, |s_d|). Parameters: match +2, mismatch −1, gap −2.
s_q = [271, 089, 314, 452, 177]
H(i,j) = max( 0,   H(i-1,j-1) + σ(t_i^q, t_j^d),   H(i-1,j)−2,   H(i,j-1)−2 )
σ(a,b) = +2 if a=b, else −1
